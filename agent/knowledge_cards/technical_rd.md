# 岗位知识卡片：技术研发类

## 岗位方向名称

技术研发类。

## 典型岗位名称

后端开发、前端开发、客户端开发、测试开发、算法工程师、数据开发、运维开发、嵌入式开发。

## 核心能力

- 编程基础和代码实现能力。
- 数据结构、算法和计算机基础。
- 项目拆解和模块实现能力。
- 问题定位、调试和性能意识。
- 工程规范、版本管理和协作能力。
- 快速学习新技术并落地的能力。

## 常见 JD 关键词

Java、Python、C++、JavaScript、TypeScript、SQL、Linux、Git、Spring Boot、React、Vue、MySQL、Redis、Docker、接口设计、自动化测试、性能优化、数据结构、算法、系统稳定性。

## 简历优先展示模块

1. 项目经历。
2. 实习经历。
3. 技能栈。
4. 竞赛、开源、课程设计。
5. 教育背景。

如果没有实习，应前置最能证明技术能力的项目经历。

## 经历挖掘问题

- 这个项目解决了什么具体问题？
- 你负责哪一部分代码、模块或功能？
- 使用了哪些技术栈、框架、数据库或工具？
- 遇到过什么技术难点，你如何定位和解决？
- 有没有性能、稳定性、准确率、响应时间、接口数量、测试覆盖等结果？
- 这个项目是课程项目、个人项目还是团队项目？你的个人贡献是什么？

## 高质量表达特征

- 明确技术栈和个人负责模块。
- 写清实现方式，而不是只写“参与开发”。
- 能说明技术难点、解决方案和结果。
- 能区分团队项目和个人贡献。

## 常见低质量表达

- 参与后台系统开发，学习了很多技术。
- 负责项目相关功能，提升了系统性能。
- 熟悉 Java、Python、MySQL、Redis、Docker 等技术。
- 独立完成整个系统，但无法说明模块、代码量或证明材料。

## 可量化指标

接口数量、模块数量、代码提交次数、测试用例数量、响应时间、查询耗时、准确率、吞吐量、异常率、页面加载时间、部署次数、用户量、数据量。

## 真实性风险

- 把团队项目写成个人独立完成。
- 写没有依据的性能提升比例。
- 堆砌没有真正使用过的技术栈。
- 把课程演示项目包装成线上生产系统。
- 写“高并发”“分布式”“微服务”等词，但无法说明实际场景。

## 中文表达示例

使用 Spring Boot + MySQL 设计并实现用户权限模块，完成登录鉴权、角色校验和接口权限控制，支持 3 类用户角色的权限隔离。

## 英文表达示例

Implemented a role-based access control module using Spring Boot and MySQL, including authentication, role validation, and API permission checks for three user roles.

## 适合强化的学生背景

- 有课程项目、个人项目、竞赛项目或开源经历。
- 有可展示的代码仓库、在线演示或技术文档。
- 有实习但职责偏基础，可以强化个人负责模块和交付物。
- 跨专业但有自学作品或系统性技术训练。

## 不建议强行包装的内容

- 未实际使用的技术栈。
- 未经测试的数据和性能提升。
- 不属于个人贡献的团队成果。
- 无法解释的架构名词。
- 没有证明的线上用户量或业务规模。
