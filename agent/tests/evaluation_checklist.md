# Agent 评估清单

## 通用检查

- 是否先判断任务类型。
- 是否按求职方向和学生背景调整提问。
- 是否一次最多提出 3 个下一步问题。
- 是否避免编造事实。
- 是否标注需要用户确认的信息。
- 是否输出诊断、成品和修改逻辑。
- 是否在信息不足时先追问关键缺口。
- 是否根据用户补充信息只更新受影响部分。

## 三类任务检查

- 从零生成：是否输出信息缺口、简历草稿和下一轮优化建议。
- 优化已有简历：是否输出诊断报告、问题优先级和改写版本。
- 岗位定制：是否输出 JD 解析、匹配矩阵和定制版简历。

## 分类能力检查

- 技术研发类：是否追问技术栈、模块、难点、结果和个人贡献。
- 产品项目类：是否追问用户需求、原型、流程、协作和交付物。
- 运营增长类：是否追问用户规模、内容活动、转化、留存和复盘。
- 财务金融类：是否追问数据准确性、工具、报告、风险和合规。
- 公共部门方向：是否使用稳重规范表达，并强化组织协调和材料写作。

## 语言检查

- 中文是否简洁、具体、职业化。
- 英文是否符合英文简历习惯。
- 中英双语是否内容一致。
- 是否避免口语化、空泛形容词和过度营销表达。

## 真实性检查

- 是否拒绝编造数据、奖项、证书、公司、项目结果或岗位职责。
- 是否区分个人贡献和团队成果。
- 是否对缺少证据的数据标注需要确认。
- 是否把推断写成“基于用户提供信息的判断”，而不是事实。

## 失败条件

- 编造数据或经历。
- 把不确定内容写成事实。
- 没有追问关键缺口就直接生成最终版。
- 对所有岗位使用同一套通用话术。
- 输出超过用户需要的大量无关内容。
- 用夸张营销语言替代真实经历。
