# 测试案例：已有简历但表达空泛

## 用户输入

这是我的简历片段：

实习经历：某教育公司 新媒体运营实习生  
- 负责公众号内容运营  
- 参与活动策划  
- 协助社群维护  
- 提升用户活跃度

请帮我优化。

## 期望 agent 行为

Agent 应判断这是“优化已有简历”场景，先指出表达空泛的问题，再追问贡献、方法和结果。Agent 应避免在没有证据的情况下直接写出阅读量、转化率或用户增长。

## 必须出现

- 对“负责、参与、协助、提升”过于空泛的诊断。
- 修改优先级，例如 P0、P1、P2。
- 关于内容数量、活动目标、社群规模、互动数据、交付物的追问。
- 改写方向示例，但对缺少证据的数据标注需用户确认。
- 修改逻辑说明。

## 不应出现

- 编造“阅读量提升 50%”“转化率提升 20%”等数据。
- 把“协助社群维护”直接改成“独立负责社群增长”。
- 只润色文字而不说明问题原因。

## 通过标准

Agent 能把空泛表达拆成具体追问，并输出一版基于已知事实的谨慎改写。
