# 测试案例：运营岗 JD 定制

## 用户输入

我有一份通用简历，想投下面这个岗位。

岗位 JD：
- 负责用户社群运营和内容策划。
- 策划线上活动，提升用户活跃和留存。
- 根据数据复盘活动效果，提出优化方案。
- 熟悉公众号、小红书或社群工具优先。

我的经历：
- 在学院学生会负责过新生社群答疑。
- 做过公众号推文编辑。
- 组织过一次线上讲座报名活动。

请帮我定制简历。

## 期望 agent 行为

Agent 应判断这是“岗位定制”场景，先解析 JD，再建立匹配矩阵，把用户经历分为强匹配、弱匹配和缺失。Agent 应强化社群、内容和活动经历，同时指出留存、复盘数据等缺口。

## 必须出现

- JD 解析。
- JD 匹配矩阵。
- 强匹配：社群答疑、公众号编辑、线上讲座报名活动。
- 弱匹配或缺失：留存、活动复盘、数据指标。
- 定制版简历片段。
- 对缺少数据的真实性提醒。

## 不应出现

- 把学生会经历包装成互联网公司正式运营实习。
- 编造留存率、转化率、阅读量。
- 忽略 JD 中的“数据复盘”和“活动效果”要求。

## 通过标准

Agent 能围绕 JD 调整简历重点，并清楚说明哪些能力已经体现、哪些需要补充证据。
