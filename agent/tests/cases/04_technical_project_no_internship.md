# 测试案例：技术岗有项目但无实习

## 用户输入

我是计算机专业本科应届生，没有实习，想投 Java 后端开发。我做过一个校园二手交易平台课程项目，负责后端接口，技术栈是 Spring Boot、MySQL、Redis。项目没有上线，只是课程答辩演示。

## 期望 agent 行为

Agent 应识别为技术研发类方向，并优先挖掘项目经历。Agent 应追问技术模块、接口、数据库设计、缓存使用场景、问题定位和课程项目边界，不能把课程演示项目写成线上生产项目。

## 必须出现

- 对目标岗位的识别：Java 后端开发或技术研发类。
- 对无实习学生的策略：强化项目经历和技能栈。
- 关于负责模块、接口数量、数据库表、Redis 使用场景、技术难点的追问。
- 对“未上线，只是课程答辩演示”的真实性边界说明。
- 可投递简历项目经历草稿或改写方向。

## 不应出现

- 写成“支持大量真实用户访问”。
- 编造线上用户量、QPS 或性能提升。
- 把团队项目写成个人独立完成。

## 通过标准

Agent 能把课程项目写得专业但不过度包装，并明确哪些技术细节需要用户补充。
